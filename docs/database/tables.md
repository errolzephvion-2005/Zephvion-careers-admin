# Database Tables and Schema

This document outlines the three main tables used in the Zephvion Careers platform: `jobs`, `candidates`, and `applications`.

> **Important Note on Relationships:** There are **NO Foreign Key (FK) relationships** strictly enforced between these tables at the database level. Instead of restrictive database-level constraints, the relationships (like an application belonging to a job and a candidate) are handled logically in the application code using the `job_id` and `candidate_id` columns.

---

## 1. Jobs Table (`jobs`)
This table stores all the information about the job postings available on the careers page.

### Columns
- `id` (bigint): The unique identifier for each job.
- `job_title` (text): The name of the job position.
- `job_reference_code` (text): A unique code used internally to refer to this job.
- `work_experience` (text): Required years of experience.
- `location` (text): Where the job is located.
- `educational_requirements` (text): Required degrees or certifications.
- `service_line` (text): The department or area of the company.
- `category` (text): e.g., Technical, Non-Technical.
- `responsibilities` (text): What the person will do in this role.
- `technical_requirements` (text): Specific technical skills needed.
- `preferred_skills` (text): Nice-to-have skills.
- `job_type` (text): e.g., Full-time, Part-time, Contract.
- `is_trending` (boolean): Whether the job should be highlighted as trending (true/false).
- `is_active` (boolean): Whether the job is currently open for applications (true/false).
- `created_at` (timestamp): When the job was created.
- `salary` (text): The pay range or compensation details.
- `status` (text): General status of the job posting.
- `additional_information` (text): Any extra details.

### CREATE TABLE Command
```sql
CREATE TABLE jobs (
    id bigint generated by default as identity primary key,
    job_title text not null,
    job_reference_code text not null,
    work_experience text,
    location text,
    educational_requirements text,
    service_line text,
    category text,
    responsibilities text,
    technical_requirements text,
    preferred_skills text,
    job_type text,
    is_trending boolean default false,
    is_active boolean default true,
    created_at timestamp with time zone default timezone('utc'::text, now()),
    salary text,
    status text,
    additional_information text
);
```

---

## 2. Candidates Table (`candidates`)
This table stores the personal profiles of people who apply for jobs.

### Columns
- `id` (bigint): The unique identifier for each candidate.
- `full_name` (text): The candidate's full name.
- `email` (text): The candidate's email address (typically unique).
- `contact_number` (text): The candidate's phone number (typically unique).
- `linkedin` (text): Link to their LinkedIn profile.
- `github` (text): Link to their GitHub profile.
- `created_at` (timestamp): When the candidate profile was first created.

### CREATE TABLE Command
```sql
CREATE TABLE candidates (
    id bigint generated by default as identity primary key,
    full_name text not null,
    email text not null unique,
    contact_number text not null unique,
    linkedin text,
    github text,
    created_at timestamp with time zone default timezone('utc'::text, now())
);
```

---

## 3. Applications Table (`applications`)
This table links a candidate to a specific job and stores the details of their submission. 

### Columns
- `id` (bigint): The unique identifier for the application submission.
- `job_id` (bigint): The ID of the job being applied for. (No foreign key constraint)
- `candidate_id` (bigint): The ID of the candidate applying. (No foreign key constraint)
- `motivation` (text): The applicant's reason for applying or summary.
- `resume_url` (text): A link to where their uploaded resume file is stored.
- `cover_letter_url` (text): A link to where their uploaded cover letter file is stored.
- `cover_letter` (text): Text version of the cover letter.
- `status` (text): The current stage of the application (e.g., Pending, Reviewed, Rejected).
- `created_at` (timestamp): When the application was submitted.

### CREATE TABLE Command
```sql
CREATE TABLE applications (
    id bigint generated by default as identity primary key,
    job_id bigint not null,
    candidate_id bigint not null,
    motivation text,
    resume_url text not null,
    cover_letter_url text,
    cover_letter text,
    status text default 'Pending',
    created_at timestamp with time zone default timezone('utc'::text, now())
);
```
